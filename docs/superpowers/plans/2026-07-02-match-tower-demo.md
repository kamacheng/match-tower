# 三消塔防 Demo 教学实施计划

> **执行模式说明：** 本计划为教学引导式——用户亲手编写全部代码，Claude 负责
> 讲解概念、引导思路、审查实现、验证结果。因此本计划刻意不包含成品代码，
> 这是用户的明确要求，不是遗漏。每课以复选框跟踪进度。

**目标：** 搭建可完整游玩的三消+塔防 demo（设计见
`docs/superpowers/specs/2026-07-02-match-tower-demo-design.md`）

**架构：** 逻辑与表现分离。BoardLogic 纯类持有棋盘真相并输出事件列表；
BoardView 按事件播放动画；Soldier/Monster 独立场景通过分组交互；
Main 持有阶段状态机与全局状态。

**技术栈：** Godot 4.7 / GDScript / Tiny Swords 素材包

## 全局约束

- 棋盘 7 列 × 8 行；每轮 5 步；玩家生命 10；胜利波数 5 —— 全部集中为常量
- 交换失败不还原，照扣步数；连锁不扣步
- 士兵一律 1 级出场，等级仅靠合成提升，封顶 3 级
- 多连奖励：4 连 → 额外金币；5 连及以上或 L/T 形 → 额外 +1 步
- 士兵与宝石一起受重力垂直下落
- 战斗单向伤害：怪物不攻击士兵
- 每课结束用中文提交一次 git

## 文件结构

| 文件 | 职责 |
|---|---|
| `assets/tiny_swords/` | 美术素材（保留原包目录结构） |
| `scenes/main.tscn` + `scripts/main.gd` | 主场景、阶段状态机、全局状态、坐标换算 |
| `scripts/game_config.gd` | 全部数值常量 |
| `scripts/board_logic.gd` | 棋盘纯逻辑类（不继承 Node） |
| `scripts/board_view.gd` | 棋盘表现层（挂在 main.tscn 的 BoardView 节点） |
| `scenes/soldier.tscn` + `scripts/soldier.gd` | 士兵（剑/弓通用，类型参数化） |
| `scenes/monster.tscn` + `scripts/monster.gd` | 怪物 |
| `scenes/arrow.tscn` + `scripts/arrow.gd` | 弓兵箭矢 |

## 关键接口约定

后续课程互相依赖的名字在此统一（签名由用户在实现时细化）：

- `BoardLogic.new(列数, 行数)`；`swap(from: Vector2i, to: Vector2i)` 返回事件数组
- 事件为字典，`type` 字段区分：消除、下落、生成士兵、合成、奖励
- 士兵场景提供 `setup(类型, 等级)`；怪物场景提供 `setup(血量, 速度)`
- 怪物加入分组 `"monsters"`，士兵索敌时按组查询
- Main 提供 `cell_to_world(cell: Vector2i) -> Vector2` 坐标换算

### 第 1 课：素材导入与战场搭建

**学习目标：** Godot 项目组织、素材导入流程、TileMap/精灵摆放

- [ ] 从 itch.io 下载 Tiny Swords 素材包，整理进 `assets/tiny_swords/`
- [ ] 了解 Godot 的导入机制（.import 文件、像素风纹理过滤设置）
- [ ] 创建 `scenes/main.tscn`，按设计文档搭出节点骨架
- [ ] 用素材铺出战场地形（草地为主，装饰随意），确定棋盘区域的屏幕位置
- [ ] 验证：运行游戏能看到地形，窗口缩放表现正常
- [ ] 提交

### 第 2 课：棋盘逻辑与铺满宝石

**学习目标：** class_name 与纯逻辑类、二维数组、逻辑与表现分离的第一次实践

- [ ] 创建 `game_config.gd`：棋盘尺寸、格子像素尺寸、宝石类型枚举等常量
- [ ] 创建 `board_logic.gd`：初始化二维数组、随机填充宝石（初盘避免天然三连）
- [ ] 创建 `board_view.gd`：读取 BoardLogic 状态，在棋盘区域铺出宝石精灵
- [ ] 验证：运行可见 7×8 宝石铺在地形上，且无初始三连；临时脚本打印棋盘验证逻辑独立可用
- [ ] 提交

### 第 3 课：交换操作与步数

**学习目标：** 输入事件处理、屏幕坐标↔格子坐标、Tween 动画

- [ ] BoardView 处理点击/拖动，换算出被点的格子
- [ ] BoardLogic 实现 swap：校验相邻性，交换数据（本课先不判定消除）
- [ ] BoardView 用 Tween 播放交换动画；实现"交换不还原"
- [ ] 步数计数：交换一次扣一步，UI 显示剩余步数（临时 Label 即可）
- [ ] 验证：能交换任意相邻宝石，步数正确扣减，动画流畅
- [ ] 提交

### 第 4 课：匹配判定与士兵生成

**学习目标：** 匹配算法、交叉匹配合并、事件列表模式、场景实例化

- [ ] BoardLogic 实现匹配检测：横竖 3+ 连线，交叉合并为一组
- [ ] swap 后执行消除：剑/弓石按规则在终点格生成士兵数据，其余格清空
- [ ] 创建士兵场景（先静态贴图），BoardView/Main 按事件实例化到对应世界坐标
- [ ] 验证：手工构造必消局面，交换后正确消除且士兵出现在被移动元素终点格
- [ ] 提交

### 第 5 课：重力下落与连锁

**学习目标：** 迭代式算法（下落→补充→再检测循环）、动画排队

- [ ] BoardLogic 实现重力：宝石与士兵一起下落，顶部补新宝石
- [ ] 稳定后重新检测匹配，循环直到无消除（连锁；连锁生成的士兵在中间格）
- [ ] BoardView 按事件顺序播放下落/消除动画，输入在结算期间锁定
- [ ] 验证：消除后无悬空；连锁能触发且不扣步；士兵会随重力下落
- [ ] 提交

### 第 6 课：士兵合成与多连奖励

**学习目标：** 在既有循环中插入新规则、规则统一带来的复用

- [ ] BoardLogic：棋盘稳定检查中加入士兵合成（3 个同类同级连线→中间格高一级），合成触发下落循环
- [ ] BoardLogic：多连奖励事件（4 连金币、5 连+/L/T 形 +1 步）
- [ ] Main 接收奖励事件更新金币/步数；等级封顶 3 级
- [ ] 验证：构造合成局面确认升级与位置正确；多连奖励数值正确
- [ ] 提交

### 第 7 课：怪物生成与推进

**学习目标：** 场景实例化、分组（group）、_process 移动、信号上报

- [ ] 创建怪物场景：贴图、血量、匀速向下移动，加入 `"monsters"` 组
- [ ] Main/Monsters 节点实现波次生成（数量按波次递增，随机横向位置）
- [ ] 越过底线：玩家生命 -1，怪物移除；生命显示到临时 UI
- [ ] 验证：怪物成波出现、匀速下行、越线扣血且被清理
- [ ] 提交

### 第 8 课：士兵索敌与攻击

**学习目标：** 距离检测、计时器、投射物、按数据区分行为

- [ ] 士兵实现索敌：按组遍历怪物，取射程内最近者
- [ ] 剑兵：近程直接扣血；弓兵：发射箭矢场景，命中扣血
- [ ] 攻击力/射程随等级变化（读 game_config 常量表）
- [ ] 怪物血量归零死亡移除
- [ ] 验证：两种士兵按各自射程攻击，怪物被击杀，等级差异可感知
- [ ] 提交

### 第 9 课：阶段状态机与胜负

**学习目标：** 状态机模式、阶段切换时序、UI 汇总

- [ ] Main 实现状态机：三消（步数归零且棋盘稳定→开战）⇄ 战斗（怪物清空→回三消，步数重置）
