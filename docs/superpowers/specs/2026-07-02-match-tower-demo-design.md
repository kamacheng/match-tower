# 三消塔防 Demo 设计文档（match tower）

日期：2026-07-02
引擎：Godot 4.7（Forward Plus）
美术：Tiny Swords 素材包（Pixel Frog，免费可商用）
开发模式：教学引导式——由 Claude 指导、用户亲手编写代码

## 1. 游戏概述

2D 俯视角三消 + 塔防。棋盘与战场是同一片空间：三消阶段玩家交换宝石，
消除处原地合成士兵；战斗阶段怪物从屏幕上方直线向下推进，士兵自动攻击，
怪物越过底线扣玩家生命。两阶段循环，撑过指定波数胜利，生命归零失败。

## 2. 三消规则

### 棋盘

- 尺寸 7 列 × 8 行（常量，可调）
- 宝石 4 种：剑石、弓石、填充石 ×2（如石块、金币石）
- 填充石消除只产出少量金币，不生成士兵，用于稀释棋盘

### 操作

- 经典交换式：点击/拖动交换相邻（上下左右）两个宝石
- **交换失败不还原**：无论是否触发消除，交换都生效且扣 1 步
- 每轮三消 5 步（常量，可调）；连锁消除不扣步
- 步数归零且棋盘稳定后，自动进入战斗阶段
- 士兵不可交换、不参与宝石匹配
- 预留扩展：对角交换、任意交换等特殊能力（本期不做）

### 消除与产出

- 匹配判定：横或竖连成一线 3 个及以上同类宝石；交叉的横竖匹配合并为一组（L/T 形）
- 剑石消除 → 原地生成 1 级持剑兵；弓石消除 → 原地生成 1 级持弓兵
- **士兵一律 1 级出场**，等级只能靠士兵合成提升
- 多连额外奖励（数值可调）：
  - 4 连 → 额外金币
  - 5 连及以上、L/T 交叉形 → 额外 +1 步
- 士兵生成位置：
  - 玩家交换触发 → 被移动元素的**终点格**
  - 连锁触发 → 匹配组**中间格**（L/T 形取交叉点格）

### 重力与连锁

- 消除/合成产生空位后，**宝石与士兵一起垂直下落**（士兵不是固定障碍）
- 顶部生成新宝石补满（新宝石只会是宝石，不会是士兵）
- 下落稳定后重新检查匹配与士兵合成，循环直到棋盘稳定（连锁）
- 预留扩展：不受重力的"固定建筑"（本期不做）

### 士兵合成

- 3 个**同类型且同等级**士兵横或竖连成一线 → 合成 1 个高一级士兵
- 合成位置在中间格，另外两格空出、触发下落
- 每次棋盘稳定后检查；合成可引发新的下落与连锁
- 等级封顶 3 级；升级提升攻击力与射程（数值搭建时调）

## 3. 战斗规则

- 怪物 1 种（哥布林），从屏幕顶部随机横向位置生成，匀速直线向下
- 每波数量与血量按波次递增（暂定第 n 波 4+2n 只，可调）
- **单向伤害**：怪物不攻击士兵，士兵无生命值
- 士兵索敌：攻击射程内最近的怪物
  - 持剑兵：射程短、伤害高，挥砍直接扣血
  - 持弓兵：射程长、伤害低，发射箭矢弹道
- 怪物越过底线 → 玩家生命 -1，怪物移除
- 本波怪物全部死亡或越线 → 回到三消阶段
- 战斗期间棋盘冻结：宝石变淡、不可操作，士兵原位作战

## 4. 整局循环

```
三消阶段 ──步数用完──▶ 战斗阶段 ──怪物清空──▶ 三消阶段（下一轮，步数重置）
                        │
                        ├─ 玩家生命 ≤ 0 ──▶ 失败
                        └─ 撑过第 5 波（可调）──▶ 胜利
```

- 玩家初始生命暂定 10（可调）
- 士兵跨波次保留
- 金币本期只累计显示，不消费（为宝箱/商店预留）

## 5. 架构（方案 A：逻辑与表现分离）

### 场景结构

```
Main (Node2D)                ← 唯一主场景，持有阶段状态机
├── Battlefield (Node2D)     ← 战场地形（纯视觉）
├── BoardView (Node2D)       ← 三消表现层：宝石精灵、动画
├── Units (Node2D)           ← 士兵容器（动态实例化）
├── Monsters (Node2D)        ← 怪物容器 + 出怪逻辑
└── UI (CanvasLayer)         ← 步数、生命、波次、金币
```

### 职责划分

- **BoardLogic（纯 GDScript 类，非节点）**：唯一持有棋盘真相。
  二维数组，每格 `{类型, 种类(宝石/士兵), 等级}`。所有规则
  （交换、匹配、生成、下落、合成、多连奖励）在此计算，
  以事件列表形式输出结果。不依赖任何节点，可独立测试。
- **BoardView（节点）**：按 BoardLogic 的事件列表播放动画
  （交换、消除、下落），并通知 Main 生成士兵。不做规则判断。
- **Soldier / Monster（独立场景 .tscn）**：士兵挂攻击计时器与索敌；
  怪物挂移动与生命值。互不引用，士兵通过分组（group）索敌。
- **Main**：阶段状态机（三消 ⇄ 战斗 → 胜/负），棋盘格坐标 ↔ 世界
  坐标换算，步数/生命/波次/金币等全局状态。

### 数据流

```
玩家输入 → BoardView（拾取格子）→ BoardLogic.swap()
        → 事件列表（消除/下落/生成/合成/奖励）
        → BoardView 播放动画 + Main 更新状态/实例化士兵
```

## 6. 教学里程碑

每个里程碑都是可运行、可验证的一课：

1. 导入 Tiny Swords、搭战场地形 —— 项目组织、TileMap/精灵
2. BoardLogic 网格 + BoardView 铺满宝石 —— 逻辑与表现分离
3. 点击交换相邻宝石（不还原规则、步数） —— 输入处理、Tween
4. 匹配判定 → 消除 → 终点格生成士兵 —— 算法落地、信号
5. 重力下落 + 补位 + 连锁 —— 迭代式算法
6. 士兵合成升级 + 多连奖励
7. 怪物生成、移动、越线扣血 —— 场景实例化、分组
8. 士兵索敌与攻击（含箭矢） —— 距离检测、计时器
9. 阶段状态机 + UI + 胜负结算
10. 打磨：动画手感、音效（可选）

## 7. 验证方式

- BoardLogic 为纯类，关键规则（匹配、下落、合成）可用临时脚本
  构造棋盘断言结果，不依赖画面
- 每个里程碑结束运行游戏人工验证该课目标
- 数值（步数、波次、伤害、射程）全部集中为常量，便于调试

## 8. 明确不做（本期）

- 宝箱元素及开箱奖励（金币消费、商店）
- 特殊交换能力（对角、任意）
- 固定不落建筑
- 怪物攻击士兵 / 士兵生命值
- 多种怪物、Boss
- 存档、关卡选择
